## 1. Reto: El número equivalente a la suma de los otros dos
Imaginemos ahora un segundo ejercicio para profundizar en la lógica condicional:

**Desafío:** Escriba un algoritmo que lea tres números y determine si uno de ellos es igual a la suma de los otros dos.

### Análisis de la condición:
Para que esto se cumpla, tenemos tres escenarios posibles. Si llamamos a los números $A$, $B$ y $C$, la condición lógica sería:

$$(A = B + C) \text{ O } (B = A + C) \text{ O } (C = A + B)$$

Si cualquiera de estas tres expresiones es Verdadera, entonces existe un número que equivale a la suma de los demás.

## 2. El problema de los mensajes ausentes
En un algoritmo simple, si la condición no se cumple, el programa termina "en silencio". Esto es una mala práctica de experiencia de usuario.

Para resolverlo, podríamos intentar poner dos bloques Si seguidos: uno para el caso positivo y otro para el negativo (invirtiendo todas las condiciones). Sin embargo, esto es ineficiente y propenso a errores lógicos. Otra opción sería usar el operador NO (NOT) para negar toda la expresión, pero sigue siendo código redundante.

Para estos casos, la solución profesional es la **Estructura de Selección Doble**.

## 3. Estructura SI-ENTONCES-SINO
La estructura de selección doble permite que el programa sepa qué hacer cuando la condición resulta ser Falsa.

### Sintaxis en Pseudocódigo:
```pseudocode
Si (condición) Entonces
    // Se ejecuta si la condición es VERDADERA
    Escribir "La condición se cumplió"
Sino
    // Se ejecuta si la condición es FALSA
    Escribir "La condición NO se cumplió"
Fin Si
```

### ¿Por qué es mejor usar Sino en lugar de dos Si seguidos?
*   **Eficiencia:** El computador solo evalúa la expresión una vez.
*   **Claridad:** El código es más fácil de leer y mantener.
*   **Exclusividad:** Garantizamos que se ejecute un bloque o el otro, pero nunca ambos al mismo tiempo.

## 4. Solución al Ejercicio: Suma de dos números
Aplicando la estructura doble, nuestro algoritmo quedaría así:

```pseudocode
Algoritmo SumaDeDos
    Variables: 
        A, B, C: Entero
Inicio
    Escribir "Ingrese tres números:"
    Leer A, B, C

    Si (A = B + C) O (B = A + C) O (C = A + B) Entonces
        Escribir "Existe un número que es la suma de los otros dos."
    Sino
        Escribir "No existe ningún número que sea la suma de los otros dos."
    Fin Si
Fin
```

## 5. Prueba de Escritorio
Hagamos una corrida en frío con los siguientes datos:

### Caso A: Usuario ingresa 5, 10, 5
1.  ¿$5 = 10 + 5$? (Falso)
2.  ¿$10 = 5 + 5$? (Verdadero)
*   **Resultado:** Entra en el primer bloque y escribe: "Existe un número..."

### Caso B: Usuario ingresa 8, 2, 2
1.  ¿$8 = 2 + 2$? (Falso)
2.  ¿$2 = 8 + 2$? (Falso)
3.  ¿$2 = 8 + 2$? (Falso)
*   **Resultado:** Como todas son falsas, el flujo salta directamente al **Sino** y escribe: "No existe ningún número..."
